# 故障排查 Skill

诊断和解决开发流程中的常见问题，提供系统化的排查步骤和解决方案。

---

## 触发条件

- 用户指令："排查问题"、"诊断错误"、"解决故障"
- 代码生成失败、测试执行失败时自动触发
- 环境检查不通过时需要排查

## 常见错误模式

### 模式 1：代码生成错误

**症状**：
- 生成代码编译失败
- 生成的代码缺少必要字段
- 类名/方法名不符合规范

**排查步骤**：
1. 检查 tech-spec.md 是否完整
2. 检查代码模板是否存在
3. 检查占位符替换是否正确
4. 检查依赖的 Rules 文件是否加载

**解决方案**：
- 补充缺失的 tech-spec 章节
- 修复代码模板中的语法错误
- 重新生成代码

### 模式 2：接口调用链错误

**症状**：
- Feign 调用失败
- 服务间调用超时
- 返回结果不符合预期

**排查步骤**：
1. 检查 Feign 接口定义与 InnerController 是否一致
2. 检查服务注册中心（Nacos）是否正常
3. 检查网络连接
4. 检查请求/响应 DTO 是否匹配

**解决方案**：
- 统一 Feign 接口和 Controller 方法签名
- 重启 Nacos 服务
- 检查防火墙设置
- 对齐 DTO 字段

### 模式 3：数据库访问错误

**症状**：
- SQL 执行失败
- 数据库连接超时
- 表结构不存在

**排查步骤**：
1. 检查数据库连接配置
2. 检查表结构是否存在
3. 检查 SQL 语法
4. 检查字段类型是否匹配

**解决方案**：
- 执行建表 SQL
- 修正数据库连接配置
- 修复 SQL 语法错误

### 模式 4：测试执行失败

**症状**：
- 测试用例断言失败
- 测试环境不可访问
- 测试数据问题

**排查步骤**：
1. 检查测试环境配置
2. 检查 Nacos 服务状态
3. 检查数据库连接
4. 检查测试数据是否正确初始化

**解决方案**：
- 初始化测试数据
- 重启测试环境服务
- 修正测试用例断言

### 模式 5：规范违规错误

**症状**：
- DoD 检查失败
- 代码审查不通过
- 命名不规范

**排查步骤**：
1. 对照 DoD 检查卡逐项检查
2. 检查命名规范
3. 检查分层架构
4. 检查接口规范

**解决方案**：
- 按 DoD 要求修改代码
- 重命名不符合规范的类/方法/字段
- 调整分层依赖关系

## 排查流程

### 步骤 1：收集信息

1. 收集错误信息和堆栈跟踪
2. 收集相关日志文件
3. 收集错误发生时的上下文

### 步骤 2：识别错误模式

根据错误信息匹配常见错误模式：
- 代码生成错误
- 接口调用链错误
- 数据库访问错误
- 测试执行失败
- 规范违规错误

### 步骤 3：执行排查步骤

按照对应错误模式的排查步骤执行：
1. 检查配置文件
2. 检查依赖服务
3. 检查代码实现
4. 检查环境状态

### 步骤 4：生成解决方案

基于排查结果生成解决方案：
1. 列出所有发现的问题
2. 提供修复建议
3. 提供预防措施

### 步骤 5：验证修复

1. 应用修复方案
2. 重新执行失败的操作
3. 验证问题是否解决
