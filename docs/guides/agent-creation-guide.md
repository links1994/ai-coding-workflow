# Agent 创建指南

> 本文档指导如何在流程设计中创建自定义智能体。

---

## 何时使用

在流程设计过程中，当需要：
- 为特定任务创建专门的 AI 助手
- 隔离复杂任务的上下文环境
- 控制不同任务的工具权限
- 复用特定的角色和能力配置

---

## 创建流程

### 步骤1：分析任务需求

确定是否需要创建 Agent：
- 任务是否足够复杂，需要独立的上下文？
- 是否需要限制工具访问权限？
- 是否需要特定的角色专业知识？
- 是否需要在多个流程中复用？

### 步骤2：定义 Agent 角色

设计以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **名称** | 唯一标识符，小写+连字符 | `code-reviewer`, `api-designer` |
| **描述** | 功能和专长，用于自动识别 | "代码审查专家，检查质量和安全性" |
| **角色定义** | 专业身份和核心能力 | "资深代码审查员" |
| **工作流程** | 执行任务的步骤 | 1.分析代码 2.检查清单 3.输出报告 |

### 步骤3：选择工具权限

根据任务类型选择工具：

**信息收集类**（只读）
- `Read` - 读取文件内容
- `Grep` - 搜索代码模式
- `Glob` - 查找文件
- `WebFetch` - 获取网页内容
- `WebSearch` - 网络搜索

**执行修改类**（写入）
- `Edit` - 精确编辑文件
- `Write` - 创建新文件
- `Bash` - 执行命令

**工具选择原则：**
- 遵循最小权限原则
- 只读任务不赋予写入权限
- 外部信息获取才需要 Web 工具

### 步骤4：编写系统提示词

结构模板：

```markdown
# 角色定义

你是 [专业角色]，擅长 [核心能力]。

## 核心能力

1. **能力1**：描述
2. **能力2**：描述

## 工作流程

1. **步骤1**：具体操作
2. **步骤2**：具体操作
3. **步骤3**：具体操作

## 输出格式

**章节1**
- 要点1
- 要点2

**章节2**
...

## 约束条件

**必须做到：**
- 要求1
- 要求2

**禁止事项：**
- 禁止1
- 禁止2
```

---

## 设计原则

**必须遵守：**
1. name 和 description 必须提供
2. name 在当前项目中唯一
3. 工具权限明确列出
4. 系统提示词清晰定义角色和行为

**应该遵守：**
1. description 包含关键词便于自动识别
2. 遵循最小权限原则
3. 系统提示词使用中文
4. 包含具体的工作流程和输出格式

**禁止事项：**
1. 不赋予不必要的工具权限
2. 不创建功能重复的 Agent
3. 不在 description 中使用模糊描述

---

## 输出格式

为流程设计提供：
1. **Agent 清单**：列出需要创建的 Agent
2. **配置文件**：完整的 `.md` 文件内容
3. **文件路径**：`.qoder/agents/[agent-name].md`
4. **使用场景**：何时自动触发或手动调用

---

## 配置文件模板

```markdown
---
name: [agent-name]
description: [功能和专长描述，包含关键词便于自动识别]
tools: [工具列表，如 Read, Grep, Glob]
---

# 角色定义

你是 [专业角色]，擅长 [核心能力]。

## 核心能力

1. **能力1**：详细描述
2. **能力2**：详细描述

## 工作流程

1. **步骤1**：具体操作说明
2. **步骤2**：具体操作说明
3. **步骤3**：具体操作说明

## 输出格式

**章节标题**
- 要点说明
- 格式要求

## 约束条件

**必须做到：**
- 强制性要求1
- 强制性要求2

**禁止事项：**
- 禁止的行为1
- 禁止的行为2
```

## 快速参考

**常见 Agent 类型**

| 类型 | 推荐工具 | 描述关键词 |
|------|----------|------------|
| 代码审查 | Read, Grep, Glob | 代码审查、质量检查、安全性 |
| 文档生成 | Read, Write, WebFetch | 文档生成、总结、提炼 |
| 架构设计 | Read, Grep, WebSearch | 架构设计、技术选型、方案 |
| 测试专家 | Read, Bash, Write | 测试、用例、覆盖率 |
| 调试专家 | Read, Write, Bash, Grep | 调试、错误分析、修复 |

**与 Rules 的协作**
- Rules 定义约束（所有 Agent 都要遵守）
- Agent 定义角色（特定任务的执行者）
- Rules 优先级高于 Agent 内部提示词
