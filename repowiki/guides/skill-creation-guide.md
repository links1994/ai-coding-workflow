# Skill 创建指南

> 本文档指导如何在流程设计中创建可复用的 Skill 模块。

---

## 何时使用

在流程设计过程中，当需要：
- 将标准化流程打包为可复用模块
- 定义特定领域的专业知识集合
- 创建可在多个 Agent 中调用的能力
- 建立团队共享的最佳实践库

---

## Skill vs Agent 选择

| 维度 | Skill | Agent |
|------|-------|-------|
| **本质** | 知识和流程集合 | 独立的 AI 智能体 |
| **上下文** | 复用主 Agent 上下文 | 独立的上下文窗口 |
| **工具** | 使用调用者的工具权限 | 独立的工具权限控制 |
| **适用** | 标准化流程、知识库 | 复杂任务、需要隔离 |

**选择原则：**
- 标准化流程 → 使用 Skill
- 需要独立上下文 → 使用 Agent
- 复杂专业任务 → 可以两者结合

---

## 创建流程

### 步骤1：识别可复用流程

分析流程中适合打包为 Skill 的部分：
- 是否重复执行？
- 是否有固定步骤？
- 是否需要专业知识？
- 是否可在不同场景复用？

### 步骤2：定义 Skill 边界

明确以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **名称** | 唯一标识，小写+连字符 | `log-analyzer`, `api-doc-gen` |
| **描述** | 功能和触发场景 | "分析日志，识别错误和性能问题" |
| **输入** | 需要的输入信息 | 日志文件路径、分析范围 |
| **输出** | 产生的结果 | 分析报告、问题列表 |
| **触发词** | 自动识别的关键词 | "分析日志"、"检查错误" |

### 步骤3：设计 Skill 内容

SKILL.md 结构：

```markdown
---
name: [skill-name]
description: [功能和触发场景描述]
---

# [技能标题]

## 何时使用

描述触发此 Skill 的场景和条件。

## 执行流程

### 步骤1：[步骤名称]
详细操作说明...

### 步骤2：[步骤名称]
详细操作说明...

### 步骤3：[步骤名称]
详细操作说明...

## 检查清单

- [ ] 检查项1
- [ ] 检查项2
- [ ] 检查项3

## 输出格式

**章节1**
- 要点1
- 要点2

**章节2**
...

## 示例

### 示例1：[场景描述]
输入：...
处理：...
输出：...

### 示例2：[场景描述]
...

## 相关资源
- [参考文档1]
- [参考文档2]
```

### 步骤4：确定作用域

| 作用域 | 路径 | 适用场景 |
|--------|------|----------|
| **项目级** | `.qoder/skills/{name}/SKILL.md` | 项目特定的流程 |
| **用户级** | `~/.qoder/skills/{name}/SKILL.md` | 个人通用的能力 |

**优先级：** 项目级 Skill 覆盖用户级同名 Skill

---

## 设计原则

**必须遵守：**
1. 文件必须命名为 `SKILL.md`
2. 必须包含有效的 YAML frontmatter
3. name 和 description 必须提供
4. 描述中必须包含触发关键词

**应该遵守：**
1. Skill 专注于单一责任
2. 提供清晰的使用场景说明
3. 包含具体的执行步骤
4. 提供使用示例
5. 避免与现有 Skill 功能重复

**禁止事项：**
1. 不创建过于复杂的嵌套指令
2. 不在用户级和项目级创建同名 Skill
3. 不创建功能模糊的 Skill

## 快速参考

**常见 Skill 类型**

| 类型 | 描述 | 触发词示例 |
|------|------|------------|
| 日志分析 | 识别错误和性能问题 | "分析日志"、"检查错误" |
| API文档 | 生成 API 文档和规范 | "生成 API 文档" |
| 代码审查 | 按清单审查代码 | "审查代码"、"检查质量" |
| 测试生成 | 生成测试用例 | "生成测试"、"写单元测试" |
| 提交规范 | 规范化提交信息 | "生成提交信息" |

**与 Agent 的配合**

```
Agent (角色定义)
    ↓ 调用
Skill (标准化流程)
    ↓ 执行
具体任务
```

示例：
- `code-reviewer` Agent 调用 `security-check` Skill 进行安全检查
- `doc-generator` Agent 调用 `api-doc-gen` Skill 生成 API 文档

**最佳实践**
1. Skill 负责"怎么做"（流程）
2. Agent 负责"谁来做"（角色）
3. Rules 负责"什么不能做"（约束）
