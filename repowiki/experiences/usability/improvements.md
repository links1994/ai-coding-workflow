# 易用性持续改进追踪

> 本文件记录 AI Coding Workflow 的易用性问题及改进计划
> 更新频率：每个 Program 完成后 review 并更新

---

## 改进原则

1. **数据驱动**：基于实际使用反馈，不凭假设
2. **小步快跑**：每次改进 1-2 个点，快速验证
3. **用户中心**：减少用户认知负担，不增加操作步骤
4. **向后兼容**：改进不破坏现有流程

---

## 反馈收集模板

每个 Program 完成后，Agent 可询问（可选）：

```markdown
## Program 反馈

- Program ID: P-YYYY-NNN
- 工作流: feature-implementation / req-decomposition / http-api-testing
- 完成时间: YYYY-MM-DD

### 顺畅度评分 (1-5)
- [ ] 5 - 非常顺畅，无需改进
- [ ] 4 - 基本顺畅，有小问题
- [ ] 3 - 一般，有些地方困惑
- [ ] 2 - 不太顺畅，经常卡住
- [ ] 1 - 很困难，难以完成

### 具体问题（可选）
1. 哪个步骤最困惑？
2. 哪个确认点感觉多余？
3. 产物是否符合预期？
4. 其他建议：
```

## 改进验证方法

每个改进完成后，通过以下方式验证效果：

1. **自我检查**：改进是否遵循原则？
2. **文档 review**：相关文档是否同步更新？
3. **实际测试**：跑一个 Program 验证改进效果
4. **用户反馈**（如有）：收集用户对改进的反馈
