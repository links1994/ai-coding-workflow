# Step: 加载原始需求

## 目的
加载用户提供的原始需求描述，不做任何预处理，接受任意形式的模糊输入。

## 输入
- `raw_requirement`: 原始需求描述（可以是几句话、草稿文字、口述记录）

## 输出
- `raw_requirement_context`: 原始需求上下文对象

## 执行步骤

### 1. 接收原始输入

接受以下任意形式的输入：
- 一段描述性文字（"我想做一个 XXX 功能"）
- 草稿文档（Markdown/文本）
- 功能关键词列表
- 用户故事形式（"作为...我希望...以便..."）

### 2. 基础信息提取

从原始输入中尝试识别：
- **主题**：需求的核心主题是什么
- **角色**：涉及哪些用户角色（管理员/商家/买家/系统）
- **动作**：用户想要做什么操作
- **目的**：为什么要做这个功能

### 3. 模糊点标记

标记输入中明显不清晰的部分，供 decision 阶段重点澄清：
- 未提及角色 → 待澄清
- 未说明操作范围 → 待澄清
- 有歧义的描述 → 待澄清

## 输出格式

```yaml
raw_requirement_context:
  raw_text: "用户原始输入的完整文字"
  identified_topic: "需求主题（推测）"
  identified_roles: []
  identified_actions: []
  ambiguous_points:
    - "未明确说明 XXX"
    - "XXX 含义不清"
```
