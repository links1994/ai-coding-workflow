# Step: 澄清业务目标

## 目的
明确需求的业务背景、用户角色和核心使用场景，消除最基础的歧义。

## 输入
- `requirement_analysis`: 需求分析报告

## 输出
- `business_goal_answers`: 业务目标澄清结果

## 执行步骤

### 1. 展示已知信息

首先展示 research 阶段已识别到的内容，让用户确认是否正确：

> 根据你的描述，我理解这个需求的主题是「XXX」，请问理解是否正确？

### 2. 核心问题访谈（必问）

逐一提问，等待用户回答后再进入下一问：

1. **这个功能要解决什么问题？**
   - 当前的痛点或不便是什么？
   - 期望实现后有什么改善？

2. **谁在使用这个功能？**
   - 管理员（运营后台）？
   - 商家（商家端）？
   - 买家/用户（APP/小程序）？
   - 系统自动触发？

3. **典型的使用流程是什么？**
   - 用户做了什么操作？
   - 操作后看到/得到什么结果？

### 3. 补充问题（按需）

如果原始需求中未提及，继续追问：
- 是新增功能还是改造现有功能？
- 是否有参考的产品或原型图？

### 4. 阶段确认

汇总本轮回答，询问用户：
> 关于业务目标，我的理解是：[汇总]。是否还有需要补充或纠正的地方？

## 输出格式

```yaml
business_goal_answers:
  problem_statement: "解决什么问题"
  user_roles:
    - role: admin
      description: "管理员在后台进行 XXX 管理"
    - role: buyer
      description: "买家在 APP 端查看 XXX"
  typical_flow: "用户操作流程描述"
  is_new_feature: true
  reference: null
```

## 注意事项

- **一次只问一个问题**，不要堆积所有问题一次抛出
- 用户回答后先复述确认，再追问下一个
- 如果用户的回答引出新的疑问，先记录，在本轮结束后一并确认
