# Step: 生成测试报告

## 目的
生成完整测试报告。

## 输入
- `test_results`: 测试结果
- `validation_result`: 验证结果
- `test_cases`: 测试用例
- `test_plan`: 测试计划

## 输出
- `test_report`: 测试报告

## 报告内容

### 1. 执行摘要

- 用例总数/通过/失败/跳过
- 执行时长
- 通过率

### 2. 覆盖率统计

- 接口覆盖率
- 场景覆盖率
- 代码覆盖率（可选）

### 3. 失败用例详情

- 失败原因
- 请求/响应详情
- 建议修复方案

### 4. 遗留问题

- 未解决问题列表
- 风险等级
- 后续跟进计划

## 输出格式

```markdown
# 测试报告

## 1. 执行摘要

| 指标 | 数值 |
|------|------|
| 用例总数 | 20 |
| 通过 | 18 |
| 失败 | 2 |
| 跳过 | 0 |
| 通过率 | 90% |

## 2. 优先级统计

| 优先级 | 总数 | 通过 | 失败 |
|--------|------|------|------|
| P0 | 10 | 10 | 0 |
| P1 | 8 | 6 | 2 |
| P2 | 2 | 2 | 0 |

## 3. 失败用例详情

### TC005 - 创建{Name}-参数超长

- 失败原因: 未校验参数长度
- 建议: 添加 @Size 注解

## 4. 结论

测试通过，可进入下一阶段。
```
