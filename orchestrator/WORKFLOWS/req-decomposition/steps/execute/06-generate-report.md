# Step: 生成拆分报告

## 目的
生成需求拆分报告。

## 输入
- `requirements_tree`: 需求树
- `dependency_graph`: 依赖关系图
- `execution_plan`: 执行计划

## 输出
- `decomposition_report`: 拆分报告

## 报告内容

### 1. 需求全景扫描

- 功能需求列表
- 非功能需求列表

### 2. 业务域划分

- 业务域映射表
- 服务归属关系

### 3. 需求树

- YAML 格式的需求树（可信源）

### 4. 依赖关系图

- Mermaid 格式的依赖图

### 5. 批次执行计划

- 批次列表
- 每个批次的 Feature
- 依赖关系

### 6. 关键决策记录

- 拆分策略选择
- 重要决策说明
